Working with GitHub

Our class uses GitHub as a repository for our html files, then we use its GitHub Pages function to display the work as public webpages. This involves a few steps you'll need to take:

  1. Create a repository and enable it for GitHub Pages display.
  2. Draft your html (and eventually CSS) in VS Code, then submit it to your GitHub repo.
  3. View your GitHub content in the browser.

Create a Repo

If you already have a personal repository you will use, skip this step.

  1. Login to your GitHub account.
  2. Click your profile icon at the upper right and select Your Repositories.
  3. Click the New CTA (call-to-action, also known as a "button") to create a new repository.
  4. Enter a name for your repository — all lower-case, no spaces!
  5. Click the Create Repository CTA.

Enable GitHub Pages Display

  1. Open your repo, as in step 2 above, then select the Settings tab.
  2. Scroll down to the GitHub Pages heading, then confirm that your master branch is selected as the site Data Source. If not, follow instructions here to make that selection now.
  3. Copy the GitHub Pages address above the data source selector to a browser window (the browser will show a 404 error at this point) and bookmark it. This will be the start of any of your GitHub Pages display.
  4. Close Settings, then select the create new page/file link or CTA.
  5. Enter this filename for your new page: index.html, then enter some text for you page, something like "This will be the base index.html page."
  6. Scroll down and select the Commit changes CTA.
  7. Edit the URL in your browser window address bar to add /index.html and press Enter.
    Your URL should be formed like this: https://[your-GitHub-user-id].github.io/[your-repo-id]/index.html
  8. Admire your first published web page!

Create Files and Submit to GitHub

  1. Open VS Code and add the code for your new .html page and save it on your local machine.
  2. Login to your GitHub account and open your personal repository.
  3. Navigate to the GitHub folder where you want to place your .html file.
  4. Select the Upload files CTA.
  5. Drag and drop from your file explorer or finder, or click Choose your files and select one or more files from your computer's file system, then click strong>Open.
  6. Scroll down and select the Commit changes CTA.
  7. Add your filename in your browser window address bar, then click Enter.
    Your URL should be formed like this: https://[your-GitHub-user-id].github.io/[your-repo-id]/your-filename.html
  8. Admire your newly added or updated web page!

Add New Folders in GitHub

  • Login to your GitHub account and open your personal repository.
  • Navigate to the GitHub folder where you want to place your new subfolder(s).
  • In the Name Your File input, enter the new folder's name (all lower-case, no spaces!) followed by a slash (/); repeat as needed to define the folder structure you want.
  • Enter a new filename index.html after all your folder names are in place.
  • Type some text in editor window, something like "This will be the new index.html file."
  • Scroll down and select the Commit changes CTA.
  • You can now navigate to your new folder and upload files — .html, images, or others — as described above.
